# Мультикалькулятор объектов

Веб-инструмент для предварительной оценки стоимости клининговых работ (пожар, металл, фасады) для компаний Global Cleaning и 112 Cleaning. Менеджеры используют его при первичном обследовании объекта, чтобы быстро получить коммерческое предложение с разбивкой по блокам.

---

## Tech Stack

- **Vanilla HTML/CSS/JS** — никаких фреймворков, никакого npm/node
- Один файл `index.html` — всё приложение целиком, без сборки
- **localStorage** — сохранение расчётов в браузере (вкладка «АСТ кирпич_фасад»)
- `navigator.clipboard` — копирование текста для клиента
- CSS Grid — двухколоночная раскладка (форма + результат)
- Каждый калькулятор изолирован внутри `<iframe srcdoc="...">` прямо в `index.html`

---

## Project Structure

```
MultiCallculator/
└── index.html      # Весь проект: шапка с вкладками + три iframe-калькулятора
```

Проект намеренно однофайловый. Новые калькуляторы добавляются как:
1. Новая кнопка-таб в `<header>`
2. Новая `<section>` с `<iframe srcdoc="...">` в `<main>`

## Conventions

**Коммиты** — на русском языке, в свободном стиле. Примеры из истории:
- `Добавлена вкладка АСТ кирпич_фасад + сохранение расчётов`
- `Add files via upload`

**Ветки** — `feature/<название-калькулятора>` (пример: `feature/facade-calculator`). Основная ветка — `main`.

**Именование элементов внутри iframe** — id в camelCase на английском (`baseRate`, `heightCoef`, `clientText`). Лейблы — на русском.

**Коэффициенты** — значения `1.0`, `1.2`, `1.35` и т.д. передаются через `value` атрибут `<option>`. Итоговый коэф = произведение всех выбранных значений.

---

## Environment

Переменных окружения нет — проект работает как статический HTML без сервера.

Открывать напрямую в браузере: `index.html` или через локальный HTTP-сервер (если нужен `clipboard API` на `http://`).

## What Claude Should Know

Поведение продукта описано в `PRODUCT_SCENARIOS.md` — сверяйся с ним перед изменением кода и при добавлении новых фич.

**Бизнес-логика расчётов:**

- Базовая формула: `сумма = площадь × базовая_ставка × итоговый_коэффициент`
- Итоговый коэффициент — произведение всех выбранных коэффициентов сложности (высота, тип, загрязнение, доступ и т.д.)
- **Калькулятор пожара** дополнительно применяет «коммерческое сглаживание»: `commercialCoef = 1 + (fullCoef - 1) × 0.35`, чтобы не пугать клиента «сырым» произведением
- Калькулятор металла даёт диапазон цен (−10% / +12% от середины)
- Калькулятор фасада имеет опциональные блоки (укрепляющий состав, гидрофобизация, расшивка швов), включаемые чекбоксом

**Сохранение расчётов** реализовано только в калькуляторе фасада через `localStorage` с ключом `facadeCalcs`. Структура записи: `{ t, area, total, rate, text }`.

**Текст для клиента** — генерируется на лету в `<div class="clientText">` и копируется через `navigator.clipboard`. Это ключевая пользовательская ценность — не ломать без причины.

**Цветовые пороги** (предупреждения в расчёте): каждый калькулятор имеет свои пороги эффективной ставки с текстовыми подсказками менеджеру. Пороги заложены в бизнес-логику — не менять без согласования.

**Ставки по умолчанию:** пожар 2000 ₽/м² (базовая для пустой квартиры — минимальный объём), металл 750 ₽/м², фасад 300 ₽/м². В калькуляторе пожара итог растёт за счёт надбавок: наличие мебели, окон, мягкой мебели, бытовой техники — каждый фактор увеличивает стоимость отдельно.

**Синхронизация между устройствами** не нужна — localStorage достаточно.

**Деплой:** инструмент деплоится на хостинг и доступен по ссылке (не только офлайн).

**Как читать код:** весь код каждого калькулятора находится внутри строкового атрибута `srcdoc` в `index.html`. Отдельного файла для отдельного калькулятора не существует — нужно читать `index.html` целиком.

**После правки `srcdoc` — проверять экранирование:** это единственная техническая точка отказа. Одна неэкранированная `"` или `<` — и браузер обрезает атрибут, iframe отрисовывает пустую страницу.

**Значения по умолчанию в полях — не заглушки:** `calculate()` вызывается сразу при загрузке, поэтому менеджер видит готовый пример расчёта открыв страницу. Не обнулять поля «для чистоты».

## Don't

1. **Не выносить калькуляторы в отдельные файлы** без явного запроса — архитектура «один файл» сделана намеренно: проще передавать, открывать без сервера, нет зависимостей.

2. **Не добавлять npm/сборку/фреймворки** — проект статический, деплоится как есть без шага сборки. Любая зависимость от node_modules или бандлера нарушает это требование.

3. **Не изменять коэффициенты и пороговые ставки** без явного согласования с бизнесом — они отражают реальную экономику конкретных видов работ, не техническое решение.

4. **Не трогать формулу `commercialCoef`** в калькуляторе пожара — сглаживание `× 0.35` не баг, а намеренное ограничение «шока» от произведения коэффициентов при продаже клиенту.

5. **Не удалять «Скопировать текст для клиента»** и не менять структуру генерируемого текста без согласования — менеджеры копируют его напрямую в мессенджеры и email.

6. **Не добавлять глобальный JS в `index.html` для управления логикой внутри iframe** — `<iframe srcdoc>` создаёт отдельный document, скрипты из родительского контекста не видят DOM внутри фрейма. `document.getElementById('area')` снаружи вернёт `null`. Пример: добавил общую `calculate()` в шапку `index.html` — в калькуляторе металла она не работает.

7. **Не вставлять «живой» HTML внутрь `srcdoc` без экранирования** — внутри атрибута `srcdoc="..."` кавычки, угловые скобки и амперсанды обязательно экранируются как `&quot;`, `&lt;`, `&gt;`, `&amp;`. Неэкранированная кавычка обрезает атрибут — iframe отрисует пустую страницу. Это единственная техническая точка отказа всей архитектуры.

8. **Не сохранять в localStorage без `try/catch`** — в приватном режиме браузера и на некоторых хостингах запись в `localStorage` выбрасывает `SecurityError`, и без `try/catch` пользователь потеряет расчёт с необработанной ошибкой. Пример: менеджер открыл ссылку в инкогнито, нажал «Сохранить расчёт» — страница упала.

9. **Не добавлять поля без регистрации в массиве `ids`** — каждый калькулятор хранит массив `ids`, по которому навешиваются слушатели `input`/`change`. Поле, не добавленное в этот массив, не вызывает пересчёт. Пример: добавил поле `glassArea`, не внёс в `ids` — калькулятор игнорирует ввод, цифра не обновляется.
